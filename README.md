# Meta AI Generation API

## Overview
FastAPI service for generating images and videos using Meta AI.

## What's Working ✅

### Image Generation
- **Endpoint:** `POST /generate/images`
- **Status:** FULLY WORKING
- **Downloads:** Auto-saved to `downloads/images/`

**Example:**
```bash
curl -X POST "http://localhost:8000/generate/images?prompt=A+cat+in+space&num_images=2&download=true"
```

**Response:**
```json
{
  "task_id": "img_12345",
  "success": true,
  "prompt": "A cat in space",
  "image_urls": ["https://scontent...1.jpg", "https://scontent...2.jpg"],
  "downloaded_files": ["downloads/images/image_1.jpg", "downloads/images/image_2.jpg"],
  "download_dir": "downloads/images"
}
```

### Video Generation  
- **Endpoint:** `POST /generate/video`
- **Status:** KNOWN ISSUES - Meta AI animation API not responding to automation
- The video generation attempts to:
  1. Create images
  2. Click "Animate" button  
  3. Wait for video generation
  4. Download videos
- **Issue:** Meta AI's video animation doesn't trigger via browser automation

**Videos DO exist in Meta AI** (confirmed via history items with URLs like `https://video-sin6-2.xx.fbcdn.net/...`), but the animation trigger isn't working through automation.

## All API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | Health check |
| `/generate/images` | POST | Generate images (sync) |
| `/generate/images/async` | POST | Generate images (async) |
| `/generate/video` | POST | Generate video (sync) - has issues |
| `/generate/video/async` | POST | Generate video (async) - has issues |
| `/check-status/{task_id}` | GET | Check task status |
| `/download/{task_id}/{index}` | GET | Download specific file |
| `/list-downloads` | GET | List all downloaded files |

## Video Generation Issue Details

**Problem:** Video animation via Meta AI's "Animate" button doesn't produce videos through browser automation, even though:
1. The button is found and clicked successfully
2. The "Custom animate" option appears and is clicked
3. The page waits for 90+ seconds
4. No video elements appear in the DOM

**Evidence videos exist:**
- History items show video URLs like `https://video-sin6-2.xx.fbcdn.net/o1/v/t2/f2/m412/AQP5G-QhitwxNEbIEgz2fl4RSuzBnR...`
- These are real videos generated by Meta AI
- Manual generation works (per user)

**Attempted solutions:**
- Browser automation with Playwright (multiple selector strategies)
- metaai-api SDK (GraphQL errors)
- Various click methods (force=True, JavaScript clicks)
- Different waiting strategies (polling, long waits)
- Multiple URL approaches (/create, /media, /prompt/...)

**Conclusion:** Meta AI's video generation may have anti-automation protections or require a different API/approach not accessible through browser automation.
